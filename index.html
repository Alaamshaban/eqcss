What Are Element Queries?

Element Queries are a new way of writing CSS so the styles apply to individual elements on the page, instead of to the whole page as one document like in regular CSS. Element Queries can be based on more than just the width or height of the browser as well, you can change styles in a number of different situations, like how many lines of text or child elements an element contains.

With style scoping we can also choose whether the styles that are applied when the conditions are met applies to the object that meets the conditions or other objects on the page.

What is EQCSS?

EQCSS is a JavaScript plugin that extends CSS to include Element Queries, plus other features that regular CSS doesn't currently support. It is written in pure JavaScript, so it doesn't require jQuery or any other JavaScript library to be installed on your site to work.

How do I use Element Queries?

Installing EQCSS.js

In order to being using Element Queries on your project you will need to add a copy of EQCSS.js to every page where you need Element Queries to display.

  • Install EQCSS at the end of your <body> with 
    <script src="EQCSS.js"></script>
    .
  • (If you want to support IE8, place 
    <!--[if lt IE 9]><script src="EQCSS-polyfills.js"></script><![endif]-->
    beforehand)

Adding Element Queries

Now that you have EQCSS linked into your project you can begin adding Element Queries. You can include these directly in your HTML or store these in an external stylesheet.

  • Write your EQCSS code in the HTML page with 
    <script type="text/eqcss"> ... </script>
    .
    Or include it from an external file with 
    <script type="text/eqcss" src="code.eqcss"></script>
    .
  • The 
    EQCSS.apply()
    function is called on page load and on resize. Call it manually when you update the DOM.

How to write Element Queries with EQCSS

  • One EQCSS script can contain multiple queries, and a query can contain multiple conditions.
  • Queries have the following form: 
    @element {selector} and {condition} [ and {condition} ]* { {css} }
  • {css}
    can contain:
    • Any valid CSS rule. (Ex: 
      #id div { color: red }
      ).
    • $this
      to scope the css in the targeted element(s) only (Ex: 
      body $this:hover > b { color: red }
      ) 
  • {selector}
    is a CSS selector targeting one or many elements. Ex: "#id" or ".class".
  • {condition}
    is composed of a measure and a value. Examples: 
    • min-width: 500px
      : condition is true if the element's width is 500px or more.
    • min-width: 50%
      : condition is true if the element's width is 50% or more of its parent's width.
    • max-width: 500px
      : condition is true if the element's width is 500px or less.
    • max-width: 50%
      : condition is true if the element's width is 50% or less of its parent's width.
    • min-height: 500px
      : condition is true if the element's height is 500px or more. 
    • min-height: 50%
      : condition is true if the element's height is 50% or more of its parent's height. 
    • max-height: 500px
      : condition is true if the element's height is 500px or less. 
    • max-height: 50%
      : condition is true if the element's height is 50% or less of its parent's height. 
    • min-characters: 50
      : condition is true if the element contains 50 or more characters.
    • max-characters: 50
      : condition is true if the element contains 50 or less characters.
    • min-lines: 5
      : condition is true if the element's height is higher then 5 lines of text.
    • max-lines: 50
      : condition is true if the element's height is lower then 5 lines of text.
    • min-children: 50
      : condition is true if the element has 50 or more direct children.
    • max-children: 50
      : condition is true if the element has 50 or less direct children.
